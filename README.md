# IT Quest: Мисия Академия

**Концепция:** Защити сървърната стая

Локална образователна уеб игра за училищни събития. Играчът влиза в ролята на млад IT герой, преминава през 5 стаи в Кибер академията и изпълнява кратки интерактивни мисии, за да защити училищния сървър.

## Как работи story mode

Академията е под кибер тревога. Играчът вижда маршрут през сградата и отключва стаите една по една:

1. **Входна зала** - избор на силен дигитален пропуск.
2. **Пощенски коридор** - откриване на фишинг съобщение.
3. **Мрежова стая** - подреждане на връзката Компютър → Рутер → Сървър → Интернет.
4. **Логическа лаборатория** - активиране на защита с две условия.
5. **Сървърна стая** - избор на правилна команда от алгоритъм.

Играта е кратка и е планирана за около 1-2 минути, подходяща за демонстрации на лаптоп по време на събитие.

## Технологии

- Front-end: HTML, CSS, JavaScript
- Back-end: Node.js + Express
- База данни: SQLite
- Звук: Web Audio API, без външни файлове
- Всички визуални елементи са CSS/HTML, без CDN и интернет ресурси

## Стартиране

Инсталирайте зависимостите:

```bash
npm install
```

Стартирайте приложението:

```bash
npm start
```

Отворете:

```text
http://localhost:3000
```

Приложението работи локално след инсталация и не изисква интернет.

## Класация и точки

Резултатите се записват в `scores.db` чрез SQLite. Съхранява се само име/псевдоним, точки, време, титла и дата.

- Верен отговор: **+100 точки**
- Грешен отговор: **+20 точки**
- Бонус за време под 45 секунди: **+50**
- Бонус за време до 70 секунди: **+30**
- Бонус за време до 90 секунди: **+10**

Титли:

- 0-150: **Начинаещ изследовател**
- 151-300: **Дигитален помощник**
- 301-450: **Млад кибергерой**
- 451-550: **Бъдещ IT специалист**

Класацията се подрежда по най-висок резултат, после по най-кратко време, после по най-нова дата. Първите три места получават визуални значки: **Главен защитник**, **Мрежов герой**, **Кибер помощник**.

## Редакция на мисиите

Мисиите са в [public/app.js](./public/app.js) в масива `missions`.

Поддържани типове:

- `access-card`
- `email-detective`
- `network-order`
- `switches`
- `terminal-choice`

За мисия с подреждане използвайте `items` и `correctOrder`. Логиката сравнява избора с `mission.correctOrder.length`, така че може да работи с различен брой елементи.

## Смяна на администраторския код

Кодът за изчистване на класацията е server-side в [server.js](./server.js):

```js
const ADMIN_CODE = process.env.ADMIN_CODE || "academy2026";
```

Може да го смените чрез променлива на средата:

```powershell
$env:ADMIN_CODE="nov-kod"
npm start
```

## Изчистване на класацията

1. Отворете екрана **Класация**.
2. Натиснете **Админ**.
3. Въведете администраторския код.
4. Потвърдете изчистването.

Маршрутът `DELETE /api/scores` остава защитен със server-side код.

## Използване на училищно събитие

- Инсталирайте проекта предварително на лаптопа.
- Стартирайте с `npm start`.
- Отворете `http://localhost:3000`.
- Включете **Режим за събитие** за по-големи бутони и цял екран.
- Направете няколко тестови игри преди началото.
- Изчистете класацията при нужда от екрана **Класация**.

## API маршрути

Запазени са:

- `GET /`
- `GET /leaderboard`
- `GET /api/scores`
- `POST /api/scores`
- `DELETE /api/scores`

## Отстраняване на проблеми

Ако проектът не стартира заради липсващи или повредени зависимости:

1. Изтрийте `node_modules`.
2. Изтрийте `package-lock.json` само ако е необходимо.
3. Изпълнете `npm install`.
4. Изпълнете `npm start`.

На Windows PowerShell може да използвате:

```powershell
npm.cmd install
npm.cmd start
```

## Бележки

- Интерфейсът е изцяло на Български.
- Не се използват външни изображения, CDN или онлайн услуги.
- Базата `scores.db` се създава автоматично, ако липсва.
- Не се събират лични данни извън име или псевдоним.
